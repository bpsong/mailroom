Configure the base URL used in generated QR codes for packages

Why configure this?

QR codes encode the full URL to package details. In production, you should set this to your server's public URL (e.g., https://mailroom.company.local) so that QR codes work correctly when scanned from mobile devices.

If not configured: The system will use the request URL as a fallback, which may not work correctly in all network configurations.
